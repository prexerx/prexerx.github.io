文章编写规范

本文最后更新于：2020年10月24日 凌晨

本站文章书写规范参考指南。

文章开头类似如下的语法：

---
title: xxx
date:  xxx
categories: xxx
tags： xxx
layout: xxx
---

在Hexo中叫做Front-Matter，通过它你可以限定文章的一些行为。

所以本文重点说明Hexo的Front-Matter和文章书写格式

Front-Matter中必要的信息

title

文章标题，内置中文支持

date

文章撰写时间

categories

文章归档种类，一般认为，categories用来描述文章作者的知识结构树，文章应该明确属于某一个领域，哪怕它已经借鉴了其他领域的知识。

tags

文章的标签，一般认为，tags并不和categories用法一致，它被用来标识文章中具有的知识领域，并用类似网络用语中的关键字来标记出来，构成另一个利于查找的维度。

layout

文章的布局，由于撰写文章，它的值有两种：post和false，当不想展示文章时，使用false标注

banner_img

当前文章页面顶部大图，可以使用本地图片的相对路径，也可以为外站链接。

站内： banner_img: /img/site-fixed-images/example.jpg   # 对应存放在 /source/img/site-fixed-images/example.jpg
站外： banner_img: https://static.zkqiang.cn/example.jpg

NOTE: 图片大小建议压缩到 1MB 以内，否则会严重拖慢页面加载

文章摘要

显示在首页中和文章顶部的摘要信息，格式为：

摘要
<!-- more -->
正文

也可以使用Front-Matter格式：

---
title: 这是标题
excerpt: 这是摘要
---

隐藏文章

可以在site中隐藏文章，但依旧可以使用URL访问，格式如下：

---
hide: true
---

文章首页缩略图

使用Front-Matter形式配置，与banner_img配置类似，格式如下：

index_img: /img/example.jpg

文章图片引用

引用本地和外网资源图片，格式如下：

![](/img/example.jpg)
![](https://static.zkqiang.cn/example.jpg)

便签扩展

可以使用渲染引擎完成便签扩展，基本格式如下：

{% note success %}
文字 或者 `markdown` 均可
{% endnote %}

颜色样式：primary， secondary， success， danger， warning， info， light。

组图

如果想把多张图片按一定布局组合显示，你可以在 markdown 中按如下格式

{% gi total n1-n2-... %}
  ![](url)
  ![](url)
  ![](url)
  ![](url)
  ![](url)
{% endgi %}

total：图片总数量，对应中间包含的图片 url 数量
n1-n2-…：每行的图片数量，可以省略，默认单行最多 3 张图，求和必须相等于 total，否则按默认样式

例如 &#123;% gi 5 3-2 %&#125; ，代表共 5 张图，第一行 3 张图，第二行 2 张图。

脚注

添加脚注的正确方法如下：

这是一句话[^1]
[^1]: 这是对应的脚注

代码块

代码段请以```符号括起来，并在开头标注代码所属语言，可以根据喜好添加缩进，如：

# 类似这样
```python
    def func:
        pass
```
